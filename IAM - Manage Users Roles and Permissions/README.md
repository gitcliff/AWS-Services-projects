#  IAM - Manage Users, Roles and Permissions
 
# 📖 Summary

Worked on **IAM User Management in AWS**, focusing on **creating and securing an IAM admin user with full permissions for DevOps automation**.
Tasks included **creating an IAM user with programmatic access, verifying console login, and generating access keys** to solve **the challenge of securely managing AWS credentials for Jenkins and CLI automation**.
Set up **AWS IAM** to automate and secure infrastructure management, aiming to **enable seamless DevOps integrations with secure, properly permissioned users**.

---

# 🛠️ Tools Used

* **AWS IAM Console**: ✨ *For user creation, permission management, and access key generation.*
* **Jenkins** (planned): ⚙️ *Will be used later for CI/CD pipeline integration with AWS.*
* **Password Manager / Jenkins Credentials Store**: 🔐 *For secure storage of AWS secrets and keys.*

---

# 🎯 Skills Gained

* **IAM User Creation**: 💡 *Learned how to create an admin user with programmatic and console access.*
* **Permission Management**: 🔍 *Applied AdministratorAccess policy for full AWS resource control.*
* **Secure Credential Handling**: 🚀 *Generated and safely stored access keys critical for automation tools.*

---

# ⚠️ Challenges Faced

* **One-Time Secret Key Visibility**: 🛠️ *Ensured secure storage after creation, as secret access keys cannot be retrieved later.*
* **Access Verification**: 🔄 *Validated admin user console login to confirm permissions and access.*

---

# 💡 Why It Matters

This lab strengthens foundational AWS IAM expertise, demonstrating how to securely create and manage admin users crucial for automating cloud infrastructure tasks.
Mastering IAM user creation and credential management is essential for building secure, scalable DevOps pipelines integrating Jenkins with AWS services. 🚀☁️
